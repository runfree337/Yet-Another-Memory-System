**Decision**: every memory channel (memory, decision, feature, backlog) instantiates one common
entry model — one file + one index line, shared frontmatter (`ENTRY-TEMPLATE.md`), validated by
one shared library (`checks/entrylib.py`). Machine vocabulary (keys, values, severities, rule
ids, verdicts) is English and treated as a stable API. Confidence transitions are traced
(`unverified → verified` requires `ratified: <who>, <date>`).

**Why**: formalization was uneven across channels (memory had frontmatter + checks, decisions
had prose conventions nothing could verify). A single meta-schema makes every entry
mechanically checkable, cross-linkable (`links:`), stampable (`updated`), and keeps the checks
thin. Alternative ruled out: per-channel ad-hoc formats — each check reinvents its parser and
the vocabularies drift apart.

**Invariant**: an entry that no shared rule can validate does not exist — every channel check
imports `entrylib`, never redefines frontmatter parsing locally.
